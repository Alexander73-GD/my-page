I'm here to talk about my NAND template assuming that you're already familiar with logic gates and the language or that you don't care.


NAND template

This is my proudest work as of today, and probably forever, because I don't plan to touch this stupid language again. Here's a quick description of the program, but keep in mind that I'm not getting into details, I'll just simplify things to make it understandable. There are more detailed instructions on how to use this at the end.

The 3 main components are the preparation, the block and the ending. These are the names I chose after a thoughtful 3 seconds while I wrote this.

Preparation: here are the inputs of the logic gate, called A and B. By default, they're set to read user input, but they can be changed to ' ' (a blank space) or '+', depending on if you want a 0 or a 1.

Block: the fun part. Delimited beautifully with lots of dots and uppercase text, is the thing that makes the NAND work. A block is a NAND gate, so if you want more gates, you need to add more blocks to your program. It's divided in three parts: getting the first input, getting the second input, doing the operation (which is a really simple >>+<< [>[>->]]) and saving the result.

Ending: just a bit of code to save the output, 0 or 1, and show it.

Instructions for use

To make any gate, or any combination of NANDs, you should make some kind of diagram with the inputs and outputs needed (examples in NOR.bf and XOR.bf). After that, editing the template will be easier. For each step, you'll be replacing the first input (A), then the second input (B) and finally storing the result (C) at some position in the SVR. Copy the template where you want to, preferably a compiler. I highly recommend this one. Copy and paste the block as many times as NAND operations you need. Every time you encounter curly brackets (braces) you'll have to add many '>' symbols depending on the placeholder word.

There are 3 types of placeholders:

  • {POSITION} is the position of the cell in the SVR where the value you need to replace an input with is stored.
  • {OUTPUT} is the position of the cell in the SVR where the output of a NAND operation will be stored.
  • {FINAL OUTPUT} is the position of the cell in the SVR where the final output is stored.

Glossary:

  • Flag: special cell used for pointer control that makes things easier, especially when combined with the following concept.
  • Glider: a simple yet incredible idea I saw in mitxela's AI in Brainfuck video.
  • Stored Values Row (SVR): this is how I decided to call the second row of the memory in a display of 12 cells per row. This is where the first two inputs and the results of each operation will be saved.
